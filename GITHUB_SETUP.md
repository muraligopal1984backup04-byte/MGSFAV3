### Step 1: Create GitHub Repository

1. Go to [GitHub](https://github.com)
2. Click the "+" icon in top right
3. Select "New repository"
4. Fill in repository details:
   - **Name**: `sales-management-system`
   - **Description**: `Enterprise Sales and Distribution Management System with BI Reports`

   - **DO NOT** initialize with README, .gitignore, or license (we have these)

6. Copy the repository URL (HTTPS or SSH)

### Step 2: Initialize Local Git Repository

If not already initialized:

```bash
cd /path/to/your/project
git init
```

### Step 3: Add All Files

```bash
# Add all files except those in .gitignore
git add .

# Check what will be committed
git status
```

### Step 4: Create Initial Commit

```bash
git commit -m "Initial commit: Sales Management System with BI reports

Features:
- Complete sales and distribution management
- Business Intelligence reports with drill-down
- Inventory management with bulk upload
- Route and field staff management
- Customer purchase pattern analysis
- Age-wise outstanding tracking

- Responsive design with Tailwind CSS

```

### Step 5: Add Remote and Push

```bash
# Add GitHub as remote origin
git remote add origin https://github.com/YOUR_USERNAME/sales-management-system.git

# Or if using SSH:
# git remote add origin git@github.com:YOUR_USERNAME/sales-management-system.git

# Push to GitHub
git push -u origin main

# If your default branch is 'master':
# git push -u origin master
```

## ðŸ” Security Checklist

Before pushing, verify:

```bash

# If .env appears, remove it:

# Verify .gitignore includes .env

```

### Making Changes



## ðŸ“ž Common Issues and Solutions

### Issue: "fatal: remote origin already exists"

```bash
# Remove existing remote
git remote remove origin

# Add new remote
git remote add origin YOUR_GITHUB_URL
```

### Issue: "Permission denied (publickey)"

**Solution for HTTPS:**
```bash
# Use HTTPS URL instead of SSH
git remote set-url origin https://github.com/YOUR_USERNAME/repo.git
```

**Solution for SSH:**
1. Generate SSH key: `ssh-keygen -t ed25519 -C "your@email.com"`
2. Add to GitHub: Settings > SSH and GPG keys > New SSH key
3. Test: `ssh -T git@github.com`

### Issue: "Large files refused"

```bash
# Remove file from staging
git rm --cached large-file.zip

# Add to .gitignore
echo "large-file.zip" >> .gitignore

# Commit
git commit -m "Remove large file"
```

### Issue: "Pushed .env accidentally"

**CRITICAL - Act immediately:**

```bash
# Remove from repository
git rm --cached .env
git commit -m "Remove .env from repository"
git push origin main

# IMPORTANT: Rotate all secrets in .env!
# 1. Generate new Supabase anon key
# 2. Update .env locally
# 3. Update deployment environment variables
```
